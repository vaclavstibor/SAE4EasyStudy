Arm-Comparison Analytics

Within-subject Likert deltas (steered vs. baseline), attention funnel, time-on-task, search usage, order effects.

How to read the Likert delta charts

Study design. Every participant saw two phases of recommendations in random order: a baseline arm (recommender without steering controls) and a steered arm (our SAE sliders / natural-language steering). After each phase they answered the same shared Likert battery (p1a_accuracy, p2a_control, …, p5c_recommend).

Primary chart — Steered − Baseline. For each item we compute scoresteered − scorebaseline inside the same participant, then average. Positive (green) = steering lifted that dimension; negative (red) = it hurt. This is the direct hypothesis test for RQ1 (perceived control) and related questions about transparency, usability and satisfaction.

Sanity chart — Order effect (Phase 2 − Phase 1). Same paired deltas, but ordered by when the phase happened rather than which arm it was. If randomisation balanced the arms, this should hover near zero. A large non-zero value warns that phase order (fatigue, practice, carry-over) is leaking into the scores and the primary chart should be interpreted with care.
